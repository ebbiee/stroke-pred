  • avg_glucose_level ← LBXGLU (fasting)
  • bmi ← BMXBMI
  • smoking_status ← SMQ020 + SMQ040
  • stroke ← MCQ160F ("ever told you had a stroke")

❌ Dropped (not in NHANES public files)

  • work_type — NHANES has occupation in OCQ but the categories don't line up with Kaggle's Private / Self-employed / Govt_job / children / Never_worked.
  • Residence_type — urban/rural is intentionally redacted from public NHANES files to protect respondents.

Your retrained model will have 9 features instead of 11. Both dropped columns were weak predictors anyway.

🧭 Decisions made during the session

1 · Reviewed the existing project done

Two notebooks, three models, SMOTE + Youden's J threshold, SHAP plots. Best model = Logistic Regression at Recall 0.875 / Precision 0.135 / AUC 0.822. The precision number is the headline problem.

2 · Diagnosed the imbalance done

Identified three levers for precision: raise the decision threshold, drop SMOTE in favor of class_weight, and get more positive cases. Each comes with a trade-off.

3 · Evaluated external dataset options done

MIMIC (CITI training needed), UK Biobank (paid, £3–9k), Framingham (BioLINCC form), NHANES (free, instant), Kaggle clones (mostly duplicates of what you already had).

4 · Built NHANES downloader today

Discovered the CDC moved file hosting (the old wwwn.cdc.gov/Nchs/Nhanes/2017-2018/*.XPT path now returns the homepage). Found the new path: wwwn.cdc.gov/Nchs/Data/Nhanes/Public/{year}/DataFiles/.... Handled the 2017–2020 cycle's renamed marital column (DMDMARTZ instead of DMDMARTL).

5 · Successfully assembled 20,694 patients today

898 stroke positives — 3.6× more than the Kaggle file alone. If concatenated, you'd have 1,147 total positives across 25,804 patients.

⚠️ Before you combine the data

Three things to fix before pd.concat

  • Label semantics differ. Kaggle's stroke is from a hospital/registry source. NHANES MCQ160F is self-reported ("has a doctor ever told you…"). Treat them as related but not identical — and report both source labels in your features (add a source column).
  • Glucose has many NaNs. Fasting glucose (LBXGLU) was only measured on ~25–35% of NHANES participants. Don't mean-impute — either drop those rows, use total glucose from BIOPRO, or add a glucose_missing indicator.
  • Drop two columns. Remove work_type and Residence_type from your Kaggle file before concatenating, since NHANES doesn't have them. Retrain on the 9-feature schema.

🚀 What's next (pick one)

A · Just fix precision

Keep using the Kaggle data only. Drop SMOTE, switch to class_weight='balanced', sweep thresholds, plot PR curve, pick an operating point. Smallest change, fastest result.

B · Combine the two datasets

Merge NHANES + Kaggle into one 25k-row training set (with a source column), retrain LR/RF/XGB, compare. Bigger lift but real methodological work.

C · Productionize

Wrap the pickled model + threshold in a predict.py + FastAPI endpoint. Clean up the repo (README, requirements.txt, .gitignore, delete Untitled.ipynb).

My honest recommendation: do A first (1 hour of work, measurable improvement), then B (the real win), then C once the model is actually good enough to deploy.
