👥 Lever 3 — predict on a higher-risk subgroup

If the base rate in your test population is 5%, no model can have precision much above ~25% at useful recall. But if you only predict on patients likely to be at risk, the base rate goes up and the ceiling goes with it. Age is the strongest signal anyway.

🔬 Why can't we go higher? (the honest math)

There's a Bayesian ceiling baked into the data

With a 5% base rate and weakly-predictive demographic features, the model is essentially asking: "given this person's age, glucose, BP, BMI, smoking — how many similar people in the training set had strokes?" For the riskiest cluster the answer is roughly 1 in 4. That's the limit. No clever loss function, no fancier model, no more data of the same kind will move it much.

To actually break the ceiling you'd need new information, not new data:

  • Lab values NHANES does have but Kaggle doesn't — HbA1c, LDL, hsCRP, atrial fibrillation, prior TIA.
  • Imaging / EHR signals — but those need MIMIC-IV or UK Biobank.
  • Time-to-event data (when did the stroke happen?) instead of just yes/no.

A sanity check against published research

The classic Framingham Stroke Risk Profile — built on full clinical data with hundreds of stroke cases — reports AUC around 0.74–0.80 for 10-year stroke prediction. Your model is already in that range (ROC-AUC ~0.81). You're not undershooting the science. You're at the science.

✅ My recommendation, honest version

Pick one of these two operating points and call it done

A · "Screening" mode

Catch as many strokes as possible, accept false alarms.

  • Variant B + GradBoost
  • Threshold 0.10
  • Precision 15.2%, Recall 74%
  • Use when "missing a stroke is much worse than a false alarm"

B · "Top-K rank" mode ★ recommended

Don't predict yes/no. Just rank patients.

  • Variant B + GradBoost, age 50+ subgroup
  • Flag the top 2–5% by score
  • Precision 32–37%, finds ~6–15% of strokes
  • Acts like a real screening tool: doctor reviews the highest-risk cohort

Stop chasing higher precision on this dataset. You're already at the information-theoretic limit of demographic-only stroke prediction. The next 10 points of precision are locked behind different features, not different code. If you really need them, that means MIMIC-IV, UK Biobank, or an EHR-linked cohort — full stop.
